# 0001. 認証で Django Form を使わない

**ステータス**: 採用（2026-07-16）

## 背景

認証は login_id + password + TOTP の固定項目のみ。会員登録画面はなく、
入力フォームが動的に増減することもない。

## 決定

Django Form を使わず、テンプレートに素の HTML で `<input>` を書き、
view では `request.POST` から直接値を取得する。

## 理由

- 入力項目が固定かつ少数で、Form の動的フィールド管理の恩恵がない
- Form の widget に class を書くと、見た目（class）が Python 側に散る。
  class はテンプレートに集約したいという方針と相反する
- この規模ではバリデーションを自前で書く手間のほうが小さい
